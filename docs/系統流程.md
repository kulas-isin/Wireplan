# Wireplan 系統運作流程

> 貼到 [HackMD](https://hackmd.io) 即可看到下方流程圖（HackMD 內建 Mermaid，向量渲染、放大清晰）。
> 若圖沒出現：HackMD 右上「⋯」→ 確認文件為 Markdown 模式即可。

---

## 一、總覽流程圖

```mermaid
flowchart TD
    A([開啟 Wireplan 網頁]) --> B{localStorage 有資料?}
    B -- 有 --> C[載入既有專案]
    B -- 無 --> D[建立預設專案]
    C --> E[選擇 / 新增專案]
    D --> E

    E --> IN[1 . 匯入需求]
    IN --> PARSE[2 . 解析與抽取]
    PARSE --> L[結構化需求清單]
    L --> M[預覽 → 取代 / 附加]
    M --> GEN[3 . 自動產出]
    GEN --> EDIT[4 . 四大分頁檢視與編輯]
    EDIT --> STATE[5 . 狀態更新 → 存 localStorage]
    STATE -. 自動載入 .-> C
    EDIT --> OUT[6 . 交付 / 備份]
```

## 二、各階段細節

### 1. 匯入需求（Import）

```mermaid
flowchart TD
    E[選擇 / 新增專案] --> F1[上傳 Excel / CSV]
    E --> F2[上傳 PDF]
    E --> F3[貼上文字 / 載入範例]
    F1 --> G1[SheetJS xlsx 讀成表格列]
    F2 --> G2[pdf.js 抽取文字]
    F3 --> G3[文字內容]
```

- Excel / CSV：用 **SheetJS** 讀成表格列（自動略過上方標題列，抓「欄位最多」的那列當表頭）。
- PDF：用 **pdf.js** 抽取文字（掃描圖檔型 PDF 無法抽字，需改貼文字）。
- 文字 / 範例：每行一項需求，可用「：」分隔名稱與說明。

### 2. 解析與抽取（src/lib）

```mermaid
flowchart TD
    H{資料型別} -- 表格 --> I1[buildFieldMapping 自動對應欄位]
    H -- 文字 --> I2[requirementsFromText]
    I1 --> J[requirementsFromRows]
    I2 --> K[detectCategory 關鍵字自動分類]
    J --> K
    K --> L[結構化需求清單]
```

- 自動對應「功能名稱 / 說明 / 工時 / 報價 / 分類…」欄位（可手動調整）。
- `detectCategory` 依關鍵字判斷分類：登入、列表、表單、儀表板、報表、流程、金流、設定…

### 3. 自動產出（Generate）

```mermaid
flowchart LR
    L[需求清單] --> O1[generateWireframe 依分類套版型]
    L --> O2[generateFlow 整體操作流程]
    L --> O3[generateSpec 規格 Markdown]
```

| 產出 | 說明 |
| --- | --- |
| Wireframe | 依需求分類自動套對應版型的可編輯線框 |
| 規格文件 | 概述、功能總表、逐項詳述、非功能需求、異動紀錄（Markdown） |
| 流程設計 | 整體操作流程，可匯出 Markdown 與 Mermaid |

### 4. 檢視與編輯（四大分頁）

```mermaid
flowchart TD
    P1[需求分頁<br/>增刪改 / 排序 / 改分類]
    P2[Wireframe 分頁<br/>編輯元件 / 桌機行動切換]
    P3[規格文件分頁<br/>編輯 / 匯出 .md]
    P4[流程設計分頁<br/>編輯節點 / 匯出 Mermaid]
    P1 --> Q{需求異動?}
    Q -- 是 --> R[按 ↻ 重新產生 版面 / 流程]
    R --> P1
```

### 5. 狀態與持久化

```mermaid
flowchart LR
    S[dispatch action] --> T[reducer 更新 state]
    T --> U[(saveState：localStorage)]
    U -. 下次開啟自動載入 .-> S
```

所有操作都經 reducer 更新狀態並即時寫入瀏覽器 `localStorage`（單機、各裝置資料不互通）。

### 6. 交付 / 備份

- 匯出 / 匯入「專案 JSON」（跨裝置搬移、交付客戶）。
- 規格文件匯出 `.md`、流程圖匯出 `.mmd`（Mermaid）。
